### Product Goal

The frontend reproduces the main Điện máy XANH shopping experience in a local Next.js application. The implementation was driven by the original site, Chrome DevTools inspection, and user-provided visual references for purchase history, mega-menu states, Flash Sale, and chatbot states.

The requested polish scope included:

- Clear error messages and retry controls for local mock operations.
- Root error boundaries to prevent a blank-screen crash.
- Loading skeletons, empty states, transitions, and success/failure toasts.
- Vietnamese currency and date formatting.
- Input validation with contextual messages.
- Realistic Vietnamese product and service content instead of placeholder text.
- Product title and favicon.
- Explicit hover, active, focus, and disabled controls.
- Responsive behavior at mobile, tablet, and desktop breakpoints.
- Local navigation that never redirects user interactions to the original website.

### Runtime And Frontend Stack

- Next.js 16.2 App Router, React 19, and strict TypeScript.
- Tailwind CSS v4 and shadcn-compatible UI utilities.
- Lucide icons plus extracted official image assets.
- App-wide `ToastProvider`, `CartProvider`, and `ChatbotAssistant`.
- Static/SSG routes where possible; search remains query-driven.
- No production backend, checkout service, order API, or external AI service is connected. Demo operations are deterministic local simulations.

### Hydration And Blank-Screen History

The reported hydration warning showed a browser extension injecting a generated `__processed_*__` attribute into `<body>` before React hydrated. The root layout now uses `suppressHydrationWarning` on `html` and `body`. App-owned hydration differences were also removed:

- Local-storage account/cart snapshots use deterministic server snapshots.
- `useSyncExternalStore` bridges account state without a server/client branch.
- Time values used by chatbot messages are created only after user actions.
- SSR output does not depend on `Date.now()`, `Math.random()`, or browser-only conditional markup.

A hard reload after the fixes showed no browser console warning, error, or failed local request.
