# M1.1–M1.8 Air-Conditioner Advisor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver a tested Vietnamese air-conditioner advisor from synthetic catalog search through deterministic decisions, guarded multi-turn FastAPI/LangGraph execution, grounded DeepSeek explanations, and the approved `frontend-mvp/` UI.

**Architecture:** Pydantic contracts define the public boundary. Pure Python owns normalization, filtering, ranking, role truth, and display selection; provider adapters are injected at the graph boundary. LangGraph persists conversation state by `session_id`; FastAPI exposes `POST /api/v1/advisor/respond`; Next.js renders server-owned fields and never recomputes decisions.

**Tech Stack:** Python 3.12, Pydantic 2, FastAPI, LangGraph 1.x, `langgraph-checkpoint-sqlite`, OpenAI-compatible clients, Langfuse, pytest; Next.js 15, React 19, TypeScript, Tailwind CSS, Playwright, npm.

## Global Constraints

- Use only `data/aircon-m1-test-data/` for M1 proof; all 14 products are synthetic and must never be labeled as live Điện Máy XANH data.
- `AIRCON-M1-001` winners remain Best Overall `AC-M1-002`, Best Value `AC-M1-003`, Cheapest Qualified `AC-M1-006`.
- `deepseek/deepseek-v4-flash` through OpenRouter explains verified results only. GPT-5.4 Mini is prohibited. GPT-5.4 Nano remains the intent/need extractor.
- Deterministic code owns normalization, hard constraints, role ranking, role truth, alternatives, and UI deduplication.
- `aircon-m1-ranking-fixture.json` is test-only. Runtime ranking receives an injected `RankingPolicy`; no production default weights are allowed.
- Input blocks at 150 words. Output generation gets one retry, then deterministic fallback.
- The response envelope uses `data`, never `result`. The public route is `POST /api/v1/advisor/respond`.
- A product may own multiple formal roles but renders once. A useful alternative is not a fourth role.
- Cursor advances only after successful product presentation.
- Preserve unrelated dirty work. Ignore `resources/`.
- Activate one Harness story packet at a time. Every production task uses RED → GREEN, a separate task review, a durable trace, and fresh verification.
- Request `gpt-5.6-luna-high` for context investigation and `gpt-5.6-terra-high` for implementation. The current subagent API cannot enforce model selection; prompts must record the request and reports must not claim enforcement.

```python
class CatalogAdapter:
    def __init__(self, catalog_path: Path): self.catalog_path = catalog_path
    def load(self) -> list[dict[str, Any]]:
        records = json.loads(self.catalog_path.read_text(encoding="utf-8"))
        if not isinstance(records, list): raise ValueError("catalog must be a list")
        return records

def search_air_conditioners(filters, *, limit=3, cursor=0, exclude_product_ids=None, adapter):
    if not 1 <= limit <= 10 or cursor < 0: raise ValueError("invalid page request")
    records = [r for r in adapter.load() if r.get("category") == filters.category and (filters.region_code is None or r.get("region_code") == filters.region_code)]
    excluded, page, scan = set(exclude_product_ids or ()), [], min(cursor, len(records))
    while scan < len(records) and len(page) < limit:
        record = records[scan]; scan += 1
        if record.get("product_id") not in excluded: page.append(record)
    return ProductSearchResult(products=page, next_cursor=scan if scan < len(records) else None, total_candidates=len(records), has_more=scan < len(records), source_snapshot=_one_snapshot(records))
```
